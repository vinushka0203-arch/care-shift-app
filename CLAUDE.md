# 介護施設シフト管理アプリ

## プロジェクト概要

介護施設向けのシフト管理アプリを開発する。

- 対象: 介護施設の管理者(シフト作成者)と職員(シフト閲覧者)
- 主な機能(MVP): 手動でのシフト作成・編集、職員の希望休・希望勤務の提出、勤務区分管理(自動シフト生成は第2段階)
- 技術スタック: FastAPI(Python)+ SQLAlchemy + React(TypeScript / Vite)+ Tailwind CSS。開発DBは SQLite、本番は PostgreSQL
- 詳細は `docs/design.md`(設計書)を参照すること。設計を変更する場合は design.md も更新する

## 学習ログの運用ルール(重要)

このプロジェクトは学習とポートフォリオ作りを兼ねている。以下を必ず守ること。

- 機能の実装・設計判断・トラブル解決を行うたびに、`docs/learning-log.md` に追記する
- 各エントリには次の3点を必ず含める:
  1. **何をしたか** — 実装・変更した内容
  2. **なぜその実装にしたか** — 検討した選択肢と、採用した理由・却下した理由
  3. **学んだこと** — 新しく知った概念・ハマったポイント・次に活かせる知見
- 「動いたから終わり」にせず、意思決定の理由を残すことを最優先にする
- ログはあとでポートフォリオ・技術記事に転用できるよう、日本語で具体的に書く

## モデルの使い分け方針

- **設計・要件整理・コードレビュー**: メインセッション(Fable 5)で行う
- **定型的な実装作業**(仕様が固まったコードの生成、テスト追加、リファクタリングなど): サブエージェント `implementer`(Sonnet)に委譲する
- ユーザーから「実装して」と依頼された際、仕様が明確であれば `implementer` サブエージェントの利用を提案すること
- アーキテクチャに影響する判断が実装中に発生した場合は、サブエージェントに任せず本体(設計担当)で判断する
